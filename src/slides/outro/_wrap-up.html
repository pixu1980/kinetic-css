Wrapping up

We built a motion toolkit using modern CSS APIs, then combined them into small scenes.

Pros

  • Less JavaScript, fewer moving parts
  • Compositional motion: layers, timing, and reuse
  • Performance friendly primitives (transform, opacity)
  • Accessibility hooks: prefers reduced motion, focus

Cons

  • Support varies: check Baseline and use @supports
  • More CSS skills required (math, timelines, stacking)
  • Debugging motion can be tricky without good tooling
  • Some features are still evolving (syntax, edge cases)

Conclusions

Motion is not decoration. It is choreography: state, timing, and meaning. Use CSS as the primary language, then add JavaScript only when you must.
  • Start with one primitive and ship it
  • Layer techniques instead of rewriting everything
  • Keep fallbacks and reduced motion in mind
